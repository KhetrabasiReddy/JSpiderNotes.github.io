Database

SQL:-

  • SQL stands for Structured Query Language.
  • It is introduced & managed by Oracle.
  • It is a Back-end language which is used to communicate with our database.

    • Database:-

    • Database is a storage location which is used to store any kind of data.
    • The four important taks related to database are:
      1. Inserting Data
      2. Updating Data
      3. Deleting Data
      4. Fetching Data

    DBMS(Database Management System):-

    • It is a software which is used to commuicate between programmer and Database.
    • The four importnt tasks related to database like: INSERTING, DELETING, UPDATING AND FETCHING data from the database is done by the help of DBMS.

    Types of DBMS:

    • RDBMS: Relational Database Management System
    • OODBMS: Object Oriented Database Management System
    • HDBMS: Hierarchical Database Management System
    • NDBMS: Network Database Management System
    • NOSQL DBMS: No Structured Query Language Database Management System

    RDBMS:-

    • It stores the data in the form of tables.
    • It deals with rows and columns.
    • Example: Oracle, DB2, MySQL etc.

    OODBMS:

    • It is used to store the data in the form of Objects.
    • Example: Student s1 = new Student(1, "ram", 234); Student s2 = new Student(2, "laxman", 2423);

    HDBMS:

    • In HDBMS the data is stored in the form of tree.
    • In case of HDBMS data are independent means there is no interconnnection between any data.

    NDBMS:

    • It stores the data in the form of tree.
    • In case of NDBMS, Data are interconnected to each other which means any data is depend on other data.

    NoSQL DBMS:

    • It store the data in keys and values format.
    • It deals with files.
    • Ex: Mongo DB.
      •  Student = {
            id: 1,
            name: "ram",
            age: 23    
        }

    DBMS

    1. It deals with Files.
    2. It cannot support Client & Server relationship.
    3. It cannot follow "Normalization" rules.
    4. It cannot support constraints.

    RDBMS

    1. It deals with tables.
    2. It supports Client & Server relationship.
    3. It follows "Normalization" Rules.
    4. It support the constraints.

    ER Diagram(Entity Relationship Diagram):-

    • It is used to display the relationship between entities, depicting how one entity is related to other entity.
    • It follows some standard orders i.e:
      1. Entity(table): Always denoted in Square shape.
      2. Attributes(columns): Always denoted in Oval shape
      3. Relationship: It is mandatory and is denoted by Rhombus shape..

    Datatypes:

    • There are three datatypes:
      1. Number
      2. Varchar
      3. Date

    Number:

    • Number datatype is used to provide integer and double values.
    • By default number datatype will accept integer values.
    • Size is not mandatory for number datatype.
    • Single quote is not mandatory for number datatypes.
    • Syntax: number(size) Or number

    Varchar:

    • Varchar is used to provide String and character type of data.
    • Incase of varchar size is mandatory.
    • It is mandatory to declare varchar inside sigle quotes.
    • Syntax: varchar(size)

    Date:

    • Date is used to provide any dates format.
    • Incase of date size is not required.
    • Date datatype is mandatory to declare inside single quotes.
    • Syntax: date
    • Examples:
      '11-November-1999' => '11-Nov-99'
      '1-Janember-2002' => '01-Jan-02'
      '11/November/1999' => '11-Nov-99'
      '11/01/1999' => it will show error
    How to check the description of any table?
    sql>desc table_name;
